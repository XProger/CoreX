Вспомогательный функционал
  1. TUtils
  2. TStream
  3. TConfigFile

Вспомогательный функционал библиотеки представлен несколькими базовыми классами, которые оптимизированы и заточены на решение определённого круга примитивных задач: работа со строками, файлами, конфигами и т.п.

TUtils
Базовый класс, включает в себя работу со строками, функции перевода чисел в строки, и обратно. Инициализируется в единственном экземпляре - Utils.
Conv(Str: string; Def: LongInt = 0): LongInt;
Переводит строку Str в целое число и возвращает результат. В случае если перевод невозможен - возвращает значение Def.
Conv(Str: string; Def: Single = 0): Single;
Переводит строку Str в вещественное число и возвращает результат. В случае если перевод невозможен - возвращает значение Def.
Conv(Str: string; Def: Boolean = False): Boolean;
Переводит строку Str в булево значение и возвращает результат. В случае если перевод невозможен - возвращает значение Def.
Conv(Value: LongInt): string;
Переводит целочисленное Value в строку и возвращает результат.
Conv(Value: Single; Digits: LongInt = 6): string;
Переводит вещественное Value в строку и возвращает результат. Digits задаёт требуемое количество знаков после запятой.
Conv(Value: Boolean): string;
Переводит булево значение Value в строку (нижний регистр) и возвращает результат.
LowerCase(Str: string): string;
Возвращает строку Str в нижнем регистре.
Trim(Str: string): string;
Удаляет пробелы в начале и конце строки Str и возвращает получившуюся строку.
DeleteChars(Str: string; Chars: TCharSet): string;
Удаляет из строки Str определённые в множестве Chars символы и возвращает получившуюся строку.
ExtractFileDir(Path: string): string;
Удаляет все символы начиная с конца строки Path до слеша.

TStream
Поток обеспечивает удобную и унифицированную работу с файлами и памятью.
Init(Memory: Pointer; MemSize: LongInt);
Инициализирует поток закреплённый за областью памяти Memory размера MemSize. В такие потоки нельзя писать данные превышающие указанный размер.
Init(FileName: string; RW: Boolean = False);
Инициализирует поток закреплённый за файлом FileName. При RW = True, будет создан пустой файл доступный для записи, в противном случае будет создан файл доступный только для чтения.
Free;
Деинициализирует поток. В случае, файлового потока, в момент вызова будет закрыт назначенный файл.
CopyFrom(Stream: TStream);
Полностью копирует содержимое другого потока, начиная с текущей позиции.
Read(Buf; BufSize: LongInt): LongInt;
Читает из потока BufSize байт данных в Buf, и возвращает количество успешно считанных. В идеале результат должен соответствовать указанному размеру. Данная функция не предназначена для файловых потоков инициализированных на запись.
Write(Buf; BufSize: LongInt): LongInt;
Пишет в поток BufSize байт данных из Buf, и возвращает количество успешно записанных. Результат должен соответствовать указанному размеру. Не работает для файловых потоках инициализированных на чтение.
Valid: Boolean;
Флаг повествующий о успешной инициализации потока после вызова Init.
Size: LongInt;
Возвращает размер потока в байтах.
Pos: LongInt;
Указывает текущую позицию курсора в потоке. Доступно для чтения/записи, но не должно превышать Size.

Описание
CoreX - кроссплатформенная библиотека содержащая в себе всё самое необходимое для создания небольшой игры. Не исключено и использование в качестве прочного фундамента для игрового движка. Отличительной особенностью библиотеки является её малый размер на фоне достаточно высокой функциональности.

Характеристики
  • Общие
    • Паскалеязычный (Delphi и FPC)
    • Весь код содержится в одном модуле
    • Объектная архитектура кода со строгим скрытием "не нужных" функций
    • Малый размер исполняемого файла
    • Поддержка платформ: Windows, Linux, MacOS
  • Обработка ввода
    • Чтение состояний клавиш мыши и клавиатуры
    • Поддержка осевых джойстиков (геймпад, рукоять, руль)
    • Ввод текста в юникоде
  • Звук
    • Удобная в использовании и высокопроизводительная аудио подсистема
    • CD качество
    • Многопоточное проигрывание сэмплов из wav
    • Многопоточное проигрывание потокового звука из ogg
    • Трёхмерное позиционирование источника и слушателя
    • Регулировка громкости и скорости проигрывания
    • Фильтры: эхо, реверберация, допплер
  • Графика
    • OpenGL 2.0
    • AntiAliasing (MSAA)
    • Поддерживается как 2D, так и 3D рендеринг
    • Поддержка dds текстур с сжатием и предрасчитанными уровнями
    • Анизотропная фильтрация текстур
  • Спрайты
    • Удобная в подготовке и использовании система спрайтовой анимации
    • Возможность назначения произвольной геометрии спрайтам

Рекомендации
С целью повышения производительности, CoreX требует строго определённого формата хранения ресурсов. Ниже представлен набор программного обеспечения, наиболее подходящего для создания ресурсов в нужном формате:

  • GIMP с установленным DDS plugin позволит сохранять dds файлы текстур.
  • Audacity имеет возможность созранения wav файлов сэмплов нужного формата без информации о исполнителе и пр. хлама, а также конвертирует в ogg файлы потокового звука.
  • Blender с установленным Collada Plugin для подготовки 3D моделей перед передачей в конвертер.
  • Если для подготовки текстур используется Adobe Photoshop, то для работы с dds рекомендуется использовать плагин от NVIDIA.
